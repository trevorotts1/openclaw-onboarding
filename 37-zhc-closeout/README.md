# Skill 37 -- ZHC Closeout

Delivers the complete Zero-Human Company final package to the client at the end of the workforce build.

## What the Client Receives at Closeout

### 1. Command Center Link
The single `<name>.zerohumanworkforce.com` Cloudflare URL -- the client's live dashboard. Every agent knows this link because run-closeout.sh writes it into AGENTS.md + TOOLS.md at closeout time.

### 2. Notion Booklet (10 sections)
A full Notion page tree built by `scripts/create-notion-closeout.sh`:

| Section | Content |
|---------|---------|
| 1. What Is a Zero-Human Company? | Defines ZHC: owner-led, AI-executed. NOT a chatbot. |
| 2. What Is a Zero-Human Workforce? | Explains the hierarchy: Owner, CEO Agent, Dept Heads, AI Employees. |
| 3. Who Is Your AI CEO? | Dedicated block: names the agent, explains the Triad Rule, how to talk to the CEO. |
| 4. Your Workforce Structure | Embeds the full visual intelligence set (org chart + all generated images). |
| 5. Departments and Roles | Per-dept sub-pages: director title + focal point + roles list + SOP count per dept. |
| 6. How to Use Your Command Center | Walkthrough: dashboard access, Kanban columns, creating tasks, talking to dept heads, 3-check rhythm. Includes the CC URL. |
| 7. Communication Hierarchy | Owner to CEO Agent to Dept Heads to AI Employees. |
| 8. Lean Six Sigma in Your ZHC | Lean (remove waste) + Six Sigma (reduce variation) + DMAIC cycle. |
| 9. Book-to-Persona System | How the workforce decides HOW to handle every task. |
| 10. Your First 7 Days | Day-by-day action plan for week 1. |

Booklet prose is defined in `templates/booklet-content.md` (editable; not locked in shell heredocs).

### 3. Visual Intelligence Set (min 3, up to 30 images)
Generated by `scripts/generate-visual-intelligence.sh` using GPT-Image-2 (primary) via Kie.ai. Each image comes from its own written prompt template.

Mandatory images (always generated):
- Org flow chart (`templates/infographic-1-prompt.md`)
- "What Is a Zero-Human Company" infographic (`templates/img-what-is-zhc-prompt.md`)
- "How Your Zero-Human Company Works" infographic (`templates/img-how-your-zhc-works-prompt.md`)

Additional images (up to cap, default 10):
- Department overview (`templates/img-dept-overview-prompt.md`)
- SOP system (`templates/img-sop-system-prompt.md`)
- Lean Six Sigma (`templates/img-six-sigma-prompt.md`)
- Workflow diagram (`templates/infographic-2-prompt.md`)

All URLs written to `.visualIntelligenceUrls` (array) in build state. `.infographic1Url` and `.infographic2Url` also set for backward compat.

### 4. Celebration Video (8s, Kie.ai Gemini Omni, real audio, client logo)
Generated by `scripts/generate-celebration-video.sh`:
- Model: gemini-omni-video (Kie.ai), Veo fallback
- Duration: 8 seconds (minimum floor, max achievable on Gemini Omni)
- Audio: generate_audio=true on both primary and fallback paths
- Logo: reads `logo_url` from branding-questions.json; composited into video image_urls alongside the org chart
- Prompt: `templates/veo-prompt.txt` -- narrative matches the 8s duration, logo directive references client brand (not hardcoded BlackCEO)

### 5. Delivery via Telegram + Operator Summary
6-message celebration sequence (send-telegram-celebration.sh) + operator summary (send-operator-summary.sh). CC URL included in msg6.

## Auto-Fire Determinism (Skill 23 -- Skill 37)

As of v12.6.0 the Skill 23 -> Skill 37 handoff is deterministic:

**PRIMARY path:** On the HOP-4 transition (when buildCompletedAt is written and closeoutStatus=pending), `resume-workforce-build.sh` directly launches `run-closeout.sh` in-process via `nohup bash ... &`. No Telegram delivery required.

**SECONDARY path (fallback):** The existing `openclaw message send` self-ping still fires as a secondary nudge. If the gateway has Telegram-offset corruption, the in-process exec has already fired.

Same logic applies in `resume-closeout-cron.sh` -- every cron fire does both.

`run-closeout.sh` is fully idempotent -- every step guards on whether its artifact already exists. A double-fire is safe.
