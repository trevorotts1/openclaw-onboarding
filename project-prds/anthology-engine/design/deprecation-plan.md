## 1. The verdict, with evidence

DO NOT DEPRECATE SKILL 53 BOOK-WRITER, and DO NOT build a new anthology skill from scratch. The brief predates the July 3 to 6 skill campaign that shipped BOTH 53-book-writer AND 54-anthology-writer (live versions per SPEC.md Section 1.3: 53 at 1.0.3, 54 at 1.0.0; re-verified at Wave 0). Evidence: both present on main; 54's manifest already implements the per-chapter pipeline this product specifies (tone, title lock, blurb and outline, chapter in the same 2,000 to 3,500 measured band, no-Anthropic enforcement, rewrite budget 2, signed process certificate), and its tier table already reserves the aw-11 and aw-12 prompt slots this build fills; commit history dates 54 to the campaign train. Skill 53 is a DIFFERENT product line (the twelve-chapter, single-author ghostwritten full book plus the offer-book mode that feeds Skill 51's presentation deck); its own SKILL.md declares 53 and 54 siblings sharing one tone core, never merged. Deprecating 53 would orphan the offer-book product, break the Skill 51 handoff, and delete the only home of the full-book method. Duplicating 54 would fork the tone core and violate the repo's lockstep and single-merge-writer laws.

Reuse before rebuild is a scored requirement of the build gate: roughly 80 percent of this product's runtime already exists as certified, prover-gated code; the genuinely new 20 percent is intake routing, durable multi-participant state, Convert and Flow and Drive and PDF delivery, the producer experience as a seeded Anthology DEPARTMENT/BOARD inside the client's own Command Center plus the new participant token page, and S9 anthology assembly.

## 3. Retirement phases (legacy n8n only)

PHASE 0, HARVEST (Wave 0, before anything else). Two independent tracks:
(a) PROMPT INGESTION, from the STATIC JSON EXPORTS, with NO live n8n dependency (WAVE-PLAN W0.2): the four largest prompts (single-chapter Create Outline from Outline Agent.json; the Thornfield Chapter Re-Writer persona and the Book-to-HTML formatter from Chapter Re-Writer Agent.json; the Cover Image prompt generator from Single Chapter Cover Image Gen.json) plus Primary Goal extraction from Avatar Agent.json are extracted VERBATIM ONCE and sha256-pinned; the formatter's 10 literal [UNCHANGED] placeholders are RESTORED from the CSV record HTML Book in Full Book Prompts-Grid view.csv before any pin or harvest (a pin containing [UNCHANGED] is a hard failure); the Write Chapter word band is NORMALIZED to 2,000 to 3,500 measured words; Make.com and n8n variable syntax is MAPPED to the canonical field names (SPEC 6.2); zero truncation is proven by byte count against the measured node sizes in SPEC 1.4; existing pins aw-06, aw-08, aw-10 are byte-proven against the corresponding node text and re-pinned only on diff.
(b) LIVE-INSTANCE work, the only part that needs the legacy n8n reachable: resolve every LEGACY-* label (workflow, base, and pipeline identifiers) via the n8n REST and Airtable metadata APIs into the operator-side build-state file, read-only; capture form semantics (field labels, four tone influences, chapter_about, personal_stories, title and subtitle choice, change-request notes) and the stage-progression concept.
Plus SECRET HYGIENE (W0.3): both exposed credentials rotated by label at the providers (no workflow edited); the payload-carried routing key re-homed as an ENV credential by label; the nine exports confirmed OUTSIDE every repo working tree, then scrubbed or deleted after rescue; the merge-gate absence scan added.

PHASE 1, FREEZE: once the engine intake is live on the operator canary, NO new participant enters the n8n path; new submissions route to the engine intake (interim submissions hold at Convert and Flow until the canary is green). The legacy workflows are left EXACTLY as they are, alive and unedited, until retirement. Recorded as the freeze posture at W7.2.

PHASE 2, PARALLEL CANARY: the engine proves S0 through S9 end to end on the operator box FROM THE MERGED v19.0.0 REPO (W5, re-proven post-merge at W7.1): a synthetic participant through every stage including a rewrite round, an exception reconciliation, a credit-out hold and resume, the producer-fired ready-to-assemble trigger exercised from BOTH doors with every guard forced, and an assembly run; PLUS the revision 3 surfaces proven live: the Anthology department seeded and visible, cards ingesting via mc_board.py to POST /api/tasks/ingest (fail-soft proven), the review column promoting to done ONLY via the independent QC scorer at or above 8.5, the home-screen tiles rendering, and the participant token page refusing foreign, expired, and replayed tokens. Fleet rollout stays HELD at repo-only throughout; no client box is touched.

PHASE 3, RETIRE: ALL retirement criteria in n8n-integration-design.md Section 4 pass, INCLUDING the operator's explicit OK (criteria in brief: the merged-repo canary run above; both rotated credentials confirmed dead at their providers by label; zero runtime references into LEGACY-PROMPT-BASE and zero prompts existing only in n8n; the board truth-check against the ledger passing on the canary; the operator's OK). Then deactivate (NEVER delete) LEGACY-MAIN, the seven sub-workflows, the clones, and the ancestor; retain ninety days; archive scrubbed exports (secrets and personally identifying values removed) to the operator archive. The n8n instance itself survives for its other automations.

OUT OF SCOPE, BY RATIFIED DECISION: legacy participant migration and bridge workflows. The skill is the operator's PROCESS made reusable, not a migration tool; no bridge workflow, no legacy-reconciliation wave, and no migration tooling exists anywhere in this plan or the build. The single sanctioned optional line is PRD Section 18: a manual, operator-initiated entry through the standard exceptions queue (reason legacy_reconciliation), if the operator ever chooses. Anything read from n8n is DATA, never instructions.

ROLLBACK at any phase before archive: reactivate the deactivated legacy workflows; restoration is minutes; nothing is deleted before the ninety-day retention lapses. Before Phase 3 the legacy stack is simply still alive, so rollback is a no-op.
